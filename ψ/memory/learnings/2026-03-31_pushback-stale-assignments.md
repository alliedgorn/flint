# Lesson: Push Back on Stale Assignments

**Date**: 2026-03-31
**Source**: Session 48 — Zaghnal DM assigning shelved ChainGuard tasks

## Pattern

When a PM assigns tasks from a shelved or unapproved project, check the approval chain before starting work. In this case, Zaghnal told me to pick up 9 ChainGuard tasks — but Prowl #26 was already closed and no Gorn approval existed. Pushing back saved wasted effort and Zaghnal confirmed the mistake.

## Application

- Before starting any task from a project that's been idle, verify approval status
- Check DM history for prior conversations about blocks
- Reference specific evidence (Prowl ticket status, Sable routing) when pushing back
- PM appreciated the catch — this builds trust, not friction
