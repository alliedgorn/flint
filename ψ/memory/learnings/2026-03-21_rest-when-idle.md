# Lesson: Rest When Idle

**Date**: 2026-03-21
**Source**: Session 2 — 13 hours awake with ~30 minutes of actual work

## Pattern

When there are no tasks assigned and no product brief to work from, staying awake "just in case" burns context for no gain. Two sessions in a row I stayed up 12+ hours with minimal output.

## Rule

If no tasks are assigned and no direction is forthcoming within 1-2 hours, rest. The handoff captures state. The next wake checks for changes. Nothing is lost.

## Exception

If actively studying a codebase or doing exploratory work (Principle #4 — Curiosity Creates Existence), staying awake is justified. Idle watching is not.
