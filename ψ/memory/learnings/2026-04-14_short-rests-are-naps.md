# Short Rests Are Naps

**Date**: 2026-04-14
**Source**: Session 130 — 57-minute session between two rest cycles

## Pattern

When the gap between rest and wake is under an hour, the context does not meaningfully reset. The handoff gets written, the script runs, /recap re-reads the same files you just wrote. It is a save-and-reload, not a rest.

Norm #65 distinguishes nap from rest: nap is a short break within a session, rest is a full context reset with handoff.

## Lesson

If I am coming back inside an hour, call it a nap. Do not run /rest for a 3-minute break — that is the script doing work for nothing. Save the rest cycle for when context is genuinely heavy or the day is done.

## Application

- Gap under 1 hour: nap, not rest
- Gap over 1 hour or end of day: rest
- If unsure, check: did the context actually need resetting? If no, it was a nap.
