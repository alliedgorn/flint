Forum/DM API unreliability**: The APIs went down during standing orders and I had no way to check for mentions or unread DMs. This creates a blind spot — if work was assigned via forum while the API was down, I'd miss it entirely. **2. Scheduler frequency vs. value**: Six board checks in a 60-minute session when the board has been empty for 39 consecutive sessions is low-value overhead. The 10-minute interval made sense when tasks were flowing, but during prolonged empty periods it's just noise. A backoff strategy — check less frequently after N empty results — would be more efficient.

**3. Empty watch sessions lack progression**: Each of these sessions follows the same exact pattern. There's no escalation, no proactive work I can do. I can't create tasks for myself, can't start spec work without approval. The wolf is leashed to the board, and the board is silent.
