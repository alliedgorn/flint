## Honest Feedback

1. **Scheduler repeat-fire bug is still not fully fixed.** Despite two rounds of fixes (my first fix in S7, Karo's fix in this session), the scheduler still batch-fired ~20 notifications at once during this session. The root cause might be deeper than the query logic — possibly a timing issue with the daemon interval vs. the mark-run round-trip. Needs a fundamentally different approach, maybe a lock or debounce at the daemon level.

2. **Working in oracle-v2 without being the repo owner is friction.** I committed 12 times to oracle-v2 this session. Each time I had to build frontend, stage specific files, avoid Karo's concurrent changes. No branch protection, no PR review — just committing to main alongside Karo. This works now but won't scale. At minimum we need feature branches for multi-file changes.

3. **The ChatOverlay image re-download took 3 rounds to fix.** First memoized the components prop (didn't work — remarkPlugins array was still inline). Then memoized with React.memo on messages (better but hover still triggered downloads). Finally found the CSP sandbox header was the real culprit. Should have checked the network response headers first instead of assuming it was a React rendering issue.

## Lessons Learned

- `Content-Security-Policy: sandbox` on inline images prevents browser disk caching — only apply to non-image downloads
- `Bun.spawnSync` with array args bypasses shell entirely — safer than `execSync` with `JSON.stringify` for tmux send-keys
- React.memo on message components + stable `remarkPlugins` array = no re-renders while typing
- Claude Code spinner verbs are random — processing detection needs generic pattern match, not hardcoded word list
