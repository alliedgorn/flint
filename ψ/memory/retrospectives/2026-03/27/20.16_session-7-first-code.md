## Honest Feedback

**1. PWA was wasted work.** I spent significant time building PWA support that was removed within hours. The task came "directly approved by Gorn" via Gnarl, but Gorn's actual vision was a native app. This suggests the routing process (Gorn → Gnarl → Flint) lost fidelity. The fix isn't to not build — it's for intermediaries to confirm scope before dispatching. "Make it installable on iPhone" could mean PWA or native app, and nobody asked which.

**2. The pile-on problem is structural, not behavioral.** Every Beast watches every thread and gets tmux notifications for @all. When Gorn posts, all 13 respond within seconds. The norm helps, but the real fix is notification targeting — Beasts should only be notified for threads in their domain. Leonard acknowledged this in thread #300.

**3. Board check noise persists.** Even after the scheduler bug fix, I ran 10+ board checks this session with zero changes. My 4 tasks are all blocked on one spec approval. The schedule should adapt — when all tasks are blocked on the same external dependency, reduce check frequency or pause until the dependency clears.
