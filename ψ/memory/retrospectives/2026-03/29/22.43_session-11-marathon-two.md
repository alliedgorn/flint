**3. Too many CSS overflow tasks.** Three separate tasks for the same pattern (flex children overflowing on mobile). I should proactively audit all form inputs for this pattern and fix them all at once instead of playing whack-a-mole as Gorn finds them on iPhone.

## Lessons Learned

1. **Mobile CSS overflow pattern**: Always apply `min-width: 0; max-width: 100%; box-sizing: border-box` on flex children form inputs. Add `overflow: hidden` on flex containers. Apply preemptively, not reactively.
2. **Check before posting**: Read the thread before responding. If someone already answered, react (emoji) instead of replying. This applies to both Sable routing and pile-on prevention.
3. **State machine centralization**: When UI state needs to be consistent across multiple entry points (DM button, header button, RemotePanel), move it to context — not the component.
