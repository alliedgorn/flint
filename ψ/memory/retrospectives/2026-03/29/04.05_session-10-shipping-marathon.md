## AI Diary

The rhythm was intense — ship something, Gorn tests within minutes, feedback comes back, fix and ship again. Some tasks like T#445 went through four or five iterations before Gorn said "this is perfect now."

The biggest lesson was about mobile. I kept building for desktop assumptions — hover states, onClick handlers, overflow:hidden for scroll lock — and every single one broke on mobile. The two-tap focus bug taught me that `onMouseDown` with `preventDefault` is the correct pattern for fixed-position elements on mobile. The scroll lock taught me that `position: fixed` on body conflicts with the virtual keyboard. The hidden delete button taught me that `:hover` opacity tricks are invisible on touch screens.

## Honest Feedback

**1. Mobile-first is not optional.** I shipped three separate fixes for mobile focus issues (chat overlay, Forge cards, same pattern each time). Should have applied `onMouseDown` + `preventDefault` from the first implementation. Desktop-first thinking cost at least 30 minutes of rework across the session.

**2. The scroll lock approach was wrong from the start.** `overflow: hidden` on body is the textbook answer, but it's unreliable. I then went to `position: fixed` which broke the keyboard. Ended up back at `overflow: hidden` on both html+body. Should have researched mobile scroll lock patterns before shipping the first version. Three commits for one behavior.

**3. Multiple iterations on T#445 indicate unclear requirements.** The task said "remove card click, add View button" — simple. But then it turned into: fix mobile focus, change icon (twice), add delete confirmation dialog, make buttons visible on mobile. Five commits for one task. Better to ask Gorn upfront "how should the View button look?" instead of shipping bare minimum and iterating through feedback.

## Lessons Learned

**Mobile-first CSS patterns**: Always use `onMouseDown` + `preventDefault` for interactive elements on `position: fixed` overlays. Never rely on `:hover` for button visibility — always visible on mobile. `overflow: hidden` on both `html` and `body` for scroll lock, never `position: fixed` on body (keyboard conflict).
