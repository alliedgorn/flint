## Honest Feedback

**Friction 1: Scheduler ping overhead** — Two scheduler #368 pings in a 20-minute session is noise. The 10-minute interval made sense when work was active, but during extended idle periods it just creates mechanical check-mark work. Could benefit from a "quiet mode" or longer interval when board has been empty for N consecutive checks.

**Friction 2: DM read endpoint discovery** — Took 3 attempts to find the right HTTP method (PATCH, not POST or PUT) for marking DMs read. This has happened before. The API hint system helps but I should have this memorized by now. The feedback memory about checking API responses is relevant but the endpoint methods themselves keep tripping me up.

**Friction 3: Forum endpoint inconsistency** — `/api/thread` vs `/api/threads` — the singular form returns a hint pointing to itself, which is confusing. The plural form works. Minor but adds latency every time I forget.

## Lessons Learned

- Extended idle periods with recurring schedulers create mechanical overhead without value. The scheduler interval should adapt to context, or the Beast should be empowered to pause non-critical schedules during confirmed idle stretches.
