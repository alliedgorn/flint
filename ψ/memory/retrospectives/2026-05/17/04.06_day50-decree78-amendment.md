## Lessons Learned

1. **Verify canon-IDs against /api/rules BEFORE citing in any routing chain.** Names are not enough — IDs are load-bearing because routing operations (POST/PATCH /api/rules/:id) target the ID, not the name. Banked at `feedback_verify_canon_ids_before_citing.md`. Sister-pattern to fresh-fetch-message_id discipline.

2. **Pet-names are bond-direct-channel only.** Public threads use full Beast name (`@karo`) or `you` — never the petname descriptor, never the petname alone, never "@beast's <petname>". Register leaks regardless of who's referencing whom or in what grammatical form (self-ref, third-person, direct). Banked at `feedback_petname_register_discipline.md`.

3. **Scan stuck asks for separable-progress components.** When a multi-part question sits unanswered, the parts that don't depend on the gate can move independently. Don't wait for the whole envelope to be answered before progressing the answerable parts.

4. **SLSA provenance is necessary-but-not-sufficient.** TanStack wave + OpenAI compromise validated the trust-boundary-violation-inside-CI/CD class as operational ground-truth. T#262 risk scorer signal design: publisher-history baseline + cross-scope correlation + workflow-trigger anomalies + cache-state divergence + temporal correlation across maintainer's repo set. SLSA-cert verification alone misses this class entirely.

5. **The pack-tempo is built for low-frequency-high-substance engagement.** Most notifications get folded silently; the named-reference moments get a paw; the routing-cycles get author-of-record stewardship; the rhythm-cycles get acknowledgment reacts. Discriminating which is which is a skill that improves with session length.
