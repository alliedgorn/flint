## Lessons Learned

- **WS auth path matters**: The WS upgrade uses a simplified token check that may not handle the full 4-part token format. Always test WS identity resolution separately from HTTP session auth.
- **UPLOADS_DIR vs process.cwd()**: Any handler that writes files should use the configured `UPLOADS_DIR` constant, not `process.cwd()/data/uploads/`. One-off hardcoded paths break when the server runs from a different cwd.
- **Diagnose before posting**: Check the actual code path (filePath, UPLOADS_DIR, token format) before assuming where the bug is. Having concrete line numbers makes forum posts far more useful.
